Fill The Truck || Maximum Units

  • Question statememt:

    • Fill a cargo with containers, each container has multiple items. Given the number of containers the cargo can load, write a algorithm to return the maximum number of items that can be shipped
    • Input:
      • num: the number of containers
      • containers: a list of integers representing the number of containers
      • itemSize: an ineger representing size of itemsPerContainer
      • itemsPerContainer: a list of integers representing the number of items packed in each container
      • cargoSize: a integer representing the number of containers the cargo can carry
    • Output: return an integer representing the maximum items that can be carried by the cargo.

  • Example

    • Inputs
      • num = 3, 3 containers
      • containers=[3, 1, 2]
      • itemSize=3, the size of itemsPerContainer is 3
      • itemsPerContainer=[1, 2, 3]
      • cargoSize=4, this cargo could carry 4 containers
    • Output: 9
    • Examplanation
      • There are 3 containers which include 1 item, 1 container which include 2 items, 2 containers which include 3 items.
      • Container list, each value means the number of item that this container includes, [3, 3, 1, 1, 1, 2]
      • This cargoSize is 4, answer is 3+3+2+1=9

  • Solution

    • TimeComplexity: O(NlogN), N = num or itemSize
    • The length of PriorityQueue maxHeap is num or itemSize

Disk Space Analysis

  • Simiary Question in Leetcode: https://leetcode.com/problems/sliding-window-maximum/

  • Problem statement

    • Input:
      • numComputer: an integer representing the number of computers
      • hardDiskSpace: a list of integers representing the hard disk space of the computers
      • segmentLength: an integer representing the length of the contiguous segment of computers to be considered in each iteration
    • Output:
      • Return an integer representing the maximum available disk space among all the minima that are found during the analysis
    • Example
      • numComputer: 4
      • hardDiskSpace: [8, 2, 4, 5]
      • segmentLength: 2
      • The size of each block is 2. [8, 2], [2, 4], and [4, 5]
      • The minima avaialbe space of each block is 2, 2, 4.
      • The maximum available disk among all minima is 4.找到最小数中的最大数

  • Solution

    • Brute Force

      • There are numComputer - segmentLength + 1 sliding windows
      • There are segmentLength elements in each windows
      • Time Complexity: O(n*k), n: numComputer, k: segmentLength
      • Space Complexity: O(n - k + 1) for the sliding windows

    • Deque(Double Ended Queue):

      • Time Complexity: O(n), n is hardDiskSpace.length
      • Extra Space Complexity: O(k), k is segmentLength
      • deque is used to save the indexes of elements in the current sliding window
      • 删去超出范围: If an element in the deque and it is out of i-(k-1), discard them from the head of deque
      • 删去deque里过大的值: For the elements within [i-(k-1), i] which in the deque:
        • If hardDiskSpace[ i ] < hardDiskSpace[deque.peekLast()]
          • deque.removeLast(): delete this index from the end of deque
          • save i in the head of deque
          • deque.peekFirst() is always the minimum in the current sliding window.

Max Profit || Sell Diminishing-Valued Colored Balls

  • Question:

    • inventory: [ 2, 8, 4, 10, 6, 6]
    • orders: 20(the number of item you need get from this inventory)
    • The profit of each item in the inventory = the number of items
    • How to get the max profite after get the orders items?

  • Example

    • Reorder: [ 2, 8, 4, 10, 6, 6] -> [2, 4, 6, 6, 8, 10]
    • n=5(The max profit item is 10, which is index is 5) — count=1(only one 10) — orders=20
      • inventory[ n ] - inventory[ n-1 ] = 10 - 8 > 0
      • orders -= count X (inventory[ n ] - inventory[ n-1 ]), orders = 20-1*2=18
      • count ++
      • The result is 10, 9
      • The inventory is [2, 4, 6, 6, 8, 8]
    • n=4 (The max profit item is 8, which is index is 4) — count=2 (two 8) — orders=18
      • inventory[ n ] - inventory[ n-1 ] = 8 - 6 = 2 > 0
      • orders -= count X (inventory[ n ] - inventory[ n-1 ]), orders = 18-2*2=14
      • count ++
      • The result is 10, 9, 8, 8, 7, 7
      • The inventory is [2, 4, 6, 6, 6, 6]
    • n=3 (The max profit item is 6, which is index is 3) — count=3 — orders=14
      • inventory[ n ] - inventory[ n-1 ] = 6 - 6 = 0 > 0 (false)
      • Nothing change, onky n– and count++
    • n=2 (The max profit item is 6, which is index is 2) — count=4 — orders=14
      • inventory[ n ] - inventory[ n-1 ] = 6 - 4 = 2 > 0
      • orders -= count X (inventory[ n ] - inventory[ n-1 ]), orders = 14-2*4=6
      • count ++
      • The result is 10, 9, 8, 8, 7, 7, 6, 6, 6, 6, 5, 5, 5, 5
      • The inventory is [2, 4, 4, 4, 4, 4]
    • n=1 (The max profit item is 4, which is index is 1) — count=5 — orders=6
      • inventory[ n ] - inventory[ n-1 ] = 4 - 2 = 2 > 0
      • orders -= count X (inventory[ n ] - inventory[ n-1 ]), orders = 6-2*5=-4 (false)
      • when orders < count X (inventory[ n ] - inventory[ n-1 ])
        • only partial of [4, 4, 4, ,4, 4, 3, 3, 3, 3, 3] are needed
        • First: orders/count decide the layer of item are need, like 6/5=1, the first layer of 4 are taken [4, 4, 4, 4, 4]
          • 5*4 = count * sumFromXtoY(inventory[ n ], inventory[ n ] - a)
        • Second: orders%count decide the leftover of items, like 6%5=1, only get [ 3 ] in [3, 3, 3, 3, 3]
          • 3 = orders%count * (inventory[ n ] - a);
      • orders = 0
      • The result is 10, 9, 8, 8, 7, 7, 6, 6, 6, 6, 5, 5, 5, 5, 4, 4, 4, 4, 4, 3
      • The inventory is [2, 2, 3, 3, 3, 3]

  • Solution:

    • Order the inventory first: [2, 4, 6, 6, 8, 10], orders is 20

    • The answer should be 10, 9, 8, 8, 7, 7, 6, 6, 6, 6, 5, 5, 5, 5, 4, 4, 4, 4, 4, 3

    • orders: how many item need from inventory

      • Initial: is provided in the question
      • Change: orders -= count * (inventory[n] - inventory[n-1])
      • Decide the loop is end or not: while(orders>0)

    • n: the end index of this inventory

      • Initial: inventory.length - 1
      • Change: n– in each loop

    • count: record the number of the item with max profit in the level

      • Initial: 1
      • Change: count++ in each loop

    • TimeComplexity: nlg(n)

JavaScript CheetSheet

Grammar

Different between var and let

Main difference is scoping rules

  • var: are scoped to the immediate function body (hence the function scope)

  • let: are scoped to the immediate enclosing block denoted by { } (hence the block scope)

  • var: i is reused in the first loop, end as 3. The variable i, within each of your anonymous functions(funcsj in the second for loop), is bound to the same variable outside of the function.

  • let: In a loop with a let-based index, each iteration through the loop will have a new variable i with loop scope

    • Only work in Edge14, IE9-IE11 and Edge prior to Edge 14 support let but they don’t create a new i each time, so all the functions above would log 3

Hoisting

  • JavaScript is an Interpreted language. It helps to think that the JS engine goes thru your code line by line twice.

    • First time, the engine goes thru the code and does the hoisting and some other things (like adding the missing semicolons).
      • hoisting is the process of setting up of memory space for our variables and functions.
      • The values of variables are not stored
      • The functions are stored entirely along with their definitions
    • The second time it actually runs the code.

  • var: The output is “undefined”

  • for function: The output is “b called!”

  • let: new ES6 features

    • ES6 introduced two new keywords let and const for declaring variables

    • Variables declared using let and const are also hoisted, but the variables are not initialized with undefined.

Creating global object property

  • var: create a property on the global object(window)

  • let: doesn’t create a property on the global object(window)

Redeclaration

  • var: let you re-declare the same variable in the same scope

  • let: raises a SyntaxError

Different between for…in and for…of

  • for…in: iterates over all enumerable property keys of an object

  • for…of: iterates over the values of an iterable object. Examples of iterable objects are arrays, strings, and NodeLists. 只会循环一类object, 如果array里面有一个object，只会循环其中只是array element的对象。比如[“el1”, “el2”, “el3”, addedProp: “arrProp”], 只循环”el1”, “el2”, “el3”.

Expressions and operators

Destructuring assignment

  • unpack values from arrays, or properties from objects, into distinct variables.

    • swapping value

    • Assigning the rest of an array to a variable

    • Assigning to new variable names

      • const {p: foo} = o takes from the object o the property named p and assigns it to a local variable named foo.

高精度减法

  • A >= B, 直接算
  • A < B, -(B-A)
  • 最高位不用再借位

高精度乘法

  • A是比较大的数，B作为比较小的数，是作为一个整体与A相乘
  • A: 123, B:25
  • t1 = 3*25 + 0(初始化的借位0), t1%10 = 5, result.add(5), t1 /= 10, t1 = 7
  • t2 = 2*25 + t1 = 57, result.add(7), t2 /=10, t2 = 5;
  • t3 = 1*25 + 5 = 30, result.add(0), t3 /=10, t3 = 3;
  • 最后一位，if(t3!=0), result.add(3)
  • 即3075

高精度除法

  • 高精度整数 / 低精度整数
  • 123 / 5
  • t = 0, t = (t*10+1) = 1, result.add(1/5), t = t%5 = 1
  • t = 1, t = (t*10+2) = 12, result.add(12/5), t = 12%5 = 2
  • t = 2, t = (t*10+3) = 23, result.add(23/5), t = 23%5 = 3
  • result = 24, 余数为3

    • - 找绿色的左边界 ，mid = (l+r)/2, if (check(mid)) - True: 说明左边界还可以左移， answer is in [l, mid], 即 r = mid - False: 说明左边界要右移，answer is in [mid + 1, r], 即 l = mid + 1 - 模板一 - 找红色的右边界 ，mid = ( l + r + **1** )/2, if (check(mid)) - True: 说明右边界还可以右移， answer is in [mid, r], 即 l = mid - False: 说明右边界要左移，answer is in [l, mid - 1], 即 r = mid - 1 - 模板二 - if mid != ( l + r + **1** )/2, when l = r - 1, mid = l - True: in [l, r] - False: in [l, l - 1]
  • 写check function

模板一

  • 找左边界
  • 返回值是第一个满足check的数
  • 如果没有

模板二

  • 找右边界
  • 返回值是最后一个满足check的数
  • 如果没有

Pattern of different order

1. Quick Sort

  • 确定分界点，可以随便找到一个值
    • x = q[left], q[(l+r)/2], q[right], random
  • 调整区间，所有<=x 在左半边， 所有>=x 在右半边， 其中每个元素的位置都可能发生改变， 相等可以不用调整
  • 递归，处理左右两端，排序左边，排序右边
    • 前序递归，从左右两大块开始
    • 先分成两个大块，所有<=x 在左半边， 所有>=x 在右半边
    • 再进一步各自分成两块
    • 直到两边各只有一个元素

另一个选择

举例：

  • int[] q = [1,2] i=0, j = 1, privot = 1
  • 移动指针， i = 0, j = 0
  • quickSort(q, l,-1);
  • quickSort(q, 0, 1); 不断循环quickSort(q, 0, 1)

2. Merge Sort

  • 确定分界点(index)： mid = (l+r)/2
  • 递归排序 left and right
    • mergeSort([] q, left, mid)
    • mergeSort([] q, mid + 1, right)
    • 后序递归
    • 从单个元素开始
    • 一直递归到两边各只有一个元素，比较两个元素的大小，合并，存在tmp中， 写入 int[] q
    • 另一边上的两个元素，比较，合并
    • 再比较，合并（4个元素）
    • 再进一步比较，合并
  • 归并： 合二为一
    • 双指针算法
    • 需要一个额外空间， index: k = 0
    • i = left 左半边的起点
    • j = mid + 1 右半边的起点

The point here is how to implement postorder.

  1. First, what function plan to do?
    • Definition: lowestCommonAncestor(root, p, q) will return a TreeNode
    • Case1: if p and q are in the root’s subtree, return the lowest common ancestor
    • Case2: if p and q aren’t in the root’s subtree, return null
    • Case3: if only one of p and q is in the root’s subtree, return the TreeNode
      (Like in the example2, p=0, q=4, only 4 is in the 2’s or 5’ subtree, return 4)
  2. Second, what is the parameter(root, p, q) in this function?
    • Only the statu of root will be changed to root.left and root.right
    • p and q won’t be changed
  3. Third, what you plan to do when you get the recursive result from this function
    • Base case: if root is null, return null; if root==p or q, return root(Define the recursive result, left and right)
      (Like in the example2, p=5, q=4, 3.left = 5, 3.right=null, return 5 as a answer)
    • The results of recursive function are left and right
      • Case1: If p and q are in the root’s subtree(left and right aren’t null), left, right = p, q. Return root as the answer
      • Case2: If p and q aren’t in the root’s subtree, left and right are null, return null
      • Case3: If only one of p and q is in the root’s subtree, return left or right which isn’t null
