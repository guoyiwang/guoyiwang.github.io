Basic Algorithm- Sorting

Posted on 2021-01-27 In Sorting

前序，中序，后续

Pattern of different order

1. Quick Sort

  • 确定分界点，可以随便找到一个值
    • x = q[left], q[(l+r)/2], q[right], random
  • 调整区间，所有<=x 在左半边， 所有>=x 在右半边， 其中每个元素的位置都可能发生改变， 相等可以不用调整
  • 递归，处理左右两端，排序左边，排序右边
    • 前序递归，从左右两大块开始
    • 先分成两个大块，所有<=x 在左半边， 所有>=x 在右半边
    • 再进一步各自分成两块
    • 直到两边各只有一个元素

另一个选择

举例：

  • int[] q = [1,2] i=0, j = 1, privot = 1
  • 移动指针， i = 0, j = 0
  • quickSort(q, l,-1);
  • quickSort(q, 0, 1); 不断循环quickSort(q, 0, 1)

2. Merge Sort

  • 确定分界点(index)： mid = (l+r)/2
  • 递归排序 left and right
    • mergeSort([] q, left, mid)
    • mergeSort([] q, mid + 1, right)
    • 后序递归
    • 从单个元素开始
    • 一直递归到两边各只有一个元素，比较两个元素的大小，合并，存在tmp中， 写入 int[] q
    • 另一边上的两个元素，比较，合并
    • 再比较，合并（4个元素）
    • 再进一步比较，合并
  • 归并： 合二为一
    • 双指针算法
    • 需要一个额外空间， index: k = 0
    • i = left 左半边的起点
    • j = mid + 1 右半边的起点

The basic pattern of binary tree is preorder, inorder and postorder.
Inorder to write the recursive code, we need to set the signature of the function at first. Understand what should do for current node

  1. First, what function plan to do?
  2. Second, what is the parameter in this function?
  3. Third, what you plan to do when you get the recursive result from this function

Example 3 - Leetcode 236 Lowest Common Ancestor of a Binary Tree


The point here is how to implement postorder.

  1. First, what function plan to do?
    • Definition: lowestCommonAncestor(root, p, q) will return a TreeNode
    • Case1: if p and q are in the root’s subtree, return the lowest common ancestor
    • Case2: if p and q aren’t in the root’s subtree, return null
    • Case3: if only one of p and q is in the root’s subtree, return the TreeNode
      (Like in the example2, p=0, q=4, only 4 is in the 2’s or 5’ subtree, return 4)
  2. Second, what is the parameter(root, p, q) in this function?
    • Only the statu of root will be changed to root.left and root.right
    • p and q won’t be changed
  3. Third, what you plan to do when you get the recursive result from this function
    • Base case: if root is null, return null; if root==p or q, return root(Define the recursive result, left and right)
      (Like in the example2, p=5, q=4, 3.left = 5, 3.right=null, return 5 as a answer)
    • The results of recursive function are left and right
      • Case1: If p and q are in the root’s subtree(left and right aren’t null), left, right = p, q. Return root as the answer
      • Case2: If p and q aren’t in the root’s subtree, left and right are null, return null
      • Case3: If only one of p and q is in the root’s subtree, return left or right which isn’t null
