Virtual Dom

Two way to create virtual DOM

  • Only JS

  • const VDOM = React.createElement(‘xx’,{id:’xx’},’xx’)

  • JSX - JavaScript XML(Extensible Markup Language)

  • JS + XML本质是Only JS中写法的语法糖, 使用babel后转成JS同上

  • 它不是字符串, 也不是HTML/XML标签, 它最终产生的就是一个JS对象

  • 遇到 <开头的代码, 以标签的语法解析: html同名标签转换为html同名元素, 其它标签需要特别解析

  • 遇到以 { 开头的代码，以JS语法解析: 标签中的js表达式必须用{ }包含

渲染虚拟DOM(元素)

  • 语法: ReactDOM.render(virtualDOM, containerDOM)

    • React find the component of MyComponent
    • If this virtualDOM is class component, a instance of this class is created and then use render in the prorotype React.Component
    • Transforming the virtual dom from the return of the “render” into DOM node that your browser can understand and display on the screen.

  • 作用: 将虚拟DOM元素渲染到页面中的真实容器DOM中显示

  • containerDOM: 用来包含virtual dom的真实dom对象

Different between in React Component render and ReactDOM.render

  • React Component render

    • No arguments and returns the corresponding React element tree(instance of Component) for that component
    • A new element React element tree is returned from the Component.render() method based on the new props and state
    • Component.render() gets called when
      • the props or the state of a component change
      • shouldComponentUpdate() returns true.

  • ReactDOM.render

    • ReactDOM.render(element, container) method is a top-level API
      • Argument1: a React element root of an element tree
      • Argument2: a container DOM element
    • Turns that passed React element into a corresponding DOM element (tree)
    • Mounts that element(DOM element) as a child in the container DOM element
    • Before mounting any DOM elements to the container though, React performs a diff between the passed element tree (is it still React element or DOM element???) and the currently mounted DOM to determine which of the DOM nodes in the currently mounted DOM have to be updated in order to match the newly passed element tree.

“this”

  • this in the render/constructor inside the component is the instance of this Class component

  • this is undefined in new created function in class component

    • ES6, React.Component doesn’t auto bind methods to itself

  • Two way to fix it

    • Bind this in constructor

      • 新建function: changeWeather时，放在哪里？ ———— Weather的原型对象（Prototype Object: React Component）上，供实例使用

      • 由于changeWeather是作为onClick的回调，所以不是通过实例调用的，是直接调用

        • 直接调用类中的方法，但类中的方法默认开启了局部的严格模式，所以changeWeather中的this为undefined

      • 通过constructor中的bind, 新建了一个changeWeather

      • changeWeather调用几次？ ———— 点几次调几次
